<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API | Untrusted user input (LLM config parameters) |
| API → External LLM | Tenant API keys sent to external services |
| API → Database | Encrypted API keys stored in database |
| API → Redis | Cached config in Redis (contains encrypted keys) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | API key in HTTP header | mitigate | Use HTTPS only (enforced by Nginx in Phase 7), Bearer token in Authorization header |
| T-02-02 | Tampering | API key stored in database | mitigate | AES-256 encryption for apiKey field, encryption key in environment variable (not in code) |
| T-02-03 | Repudiation | No audit trail for config changes | mitigate | Log all config changes with timestamp and user context (Phase 6 adds operation logging) |
| T-02-04 | Information Disclosure | API key exposed in logs | mitigate | Never log full API key, only log first 4 and last 4 characters (e.g., "sk-****...****") |
| T-02-05 | Information Disclosure | API key in Redis cache | mitigate | Redis cache contains encrypted key only, Redis access restricted to internal network |
| T-02-06 | Denial of Service | Unlimited LLM API calls | mitigate | Phase 3 implements rate limiting (100 QPS/user), Phase 3 implements circuit breaker |
| T-02-07 | Elevation of Privilege | Tenant accessing other tenant's config | mitigate | TenantContext enforces tenant_id filtering, MyBatis-Plus TenantLineInterceptor prevents cross-tenant queries |

</threat_model>
