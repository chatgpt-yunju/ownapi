  - Retryable errors: SocketTimeoutException, ConnectException, HttpResponseCode 5xx

  - Return false for other exceptions

- Method: void sleep(long milliseconds)
  - Thread.sleep(milliseconds)
  - Catch InterruptedException and log warning

Use @Slf4j for logging. This policy will be used by all LLM services when making HTTP requests.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API → Redis | Rate limit counters and cached tokens stored in Redis |
| API → Database | LLM call logs stored with sensitive information (token counts, costs) |
| API → External LLM | Retry and circuit breaker prevent cascading failures |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | Denial of Service | Rate limit bypass via multiple API keys | mitigate | Rate limit enforced per tenant_id, not per API key; multiple keys from same tenant share quota |
| T-02-11 | Denial of Service | Circuit breaker state manipulation | accept | Circuit breaker state stored in memory, not externally accessible; state transitions logged |
| T-02-12 | Information Disclosure | LLM call logs contain token counts | mitigate | Logs stored in database with tenant_id isolation; access controlled via RBAC (Phase 1) |
| T-02-13 | Tampering | Retry logic causes duplicate API calls | mitigate | Idempotency key in request headers (future enhancement); circuit breaker prevents excessive retries |

</threat_model>
