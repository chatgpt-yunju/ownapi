Python - Anthropic SDK 
import anthropic

client = anthropic.Anthropic(
    base_url="https://api.yunjunet.cn/v1",
    api_key="sk-your-api-key"
)

# Messages API
with client.messages.stream(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[{"role": "user", "content": "你好！"}]
) as stream:
    for text in stream.text_stream:
        print(text, end="", flush=True)
Python - OpenAI SDK 
from openai import OpenAI

client = OpenAI(
    base_url="https://api.yunjunet.cn/v1",
    api_key="sk-your-api-key"
)

# Chat 补全
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "你好！"}],
    stream=True
)

for chunk in response:
    print(chunk.choices[0].delta.content, end="")
Gemini CLI / cURL

  -H "x-goog-api-key: sk-your-api-key" \

API 中转技术 8 步

入口接入、鉴权验证、限流排队、动态调度、上游调用、响应回传、容灾切换，构成完整请求处理链路。

STEP 1
入口接入

用户请求抵达中转服务器，触发后续处理流程。

STEP 2
鉴权验证

校验 API Key 有效性、套餐余额及权限等级。

STEP 3
智能限流

通过动态速率限制防止上游账号过载，平衡系统压力。

STEP 4
请求排队

请求进入优先级队列等待调度，延迟主要由排队时长决定。

STEP 5
动态调度（核心）
  • 地域优选：选择延迟最低可用区
  • 额度分配：倾斜到余额充足账号
  • 健康监测：自动隔离高风险账号
  • 模型路由：按成本与质量策略路由
STEP 6
上游调用

通过选定账号向官方 API 发起真实请求，可支持多层中转。

STEP 7
响应回传

AI 生成内容沿链路回传到用户端，可能叠加中转延迟。

STEP 8
容灾处理

自动重试、熔断机制与账号切换，保障高可用与稳定体验。
