团队为什么会切换

少一点流程，多一点交付

路由器的价值不是新奇，而是把模型层变得“足够无聊”：一个地方比较、一个地方付款、一个地方接入。

01

一个 API 连接多个提供方

保留现有客户端库，只替换模型，而不是把每个集成都重做一遍。

02

成本透明

先把真实成本结构展示清楚，再把账务细节交给控制台。

03

更适合运维与管理

模型选择、使用可见性和账户管理都集中在一个团队真正能维护的地方。

官方 API 路径

多个提供方、不同 Key、分散仪表盘，以及每个模型家族都不同的计费形态。

  • 更多准备： 每个提供方都有自己的接入流程。
  • 更多切换： 价格、额度和接口都分散在不同地方。
  • 更多维护： 模型变化时需要反复调整集成。

云路由路径

一个网关、一把 Key、一个清楚的地方先评估成本，再决定要不要切到某个模型或方案。

  • 更少准备： 一个账号更快完成首个请求。
  • 更少切换： 在一个地方比较模型。
  • 更少偏差： 价格与权限保持统一。

适用人群

这个首页刻意保持简单：信息足够用户快速判断，但又不会让第一次点击显得负担太重。

  • 开发者： 希望快速验证价值。
  • 团队： 需要共享控制与清晰用量。
  • 负责人： 关注成本、稳定性和速度。
开始使用

三步完成首个请求

把接入流程尽量缩短并做得一目了然。这也符合 derouter 风格的承诺：发现、选择、启动。

01

创建账号

登录后打开控制台，先确认账号状态，再把 Key 接入你的工作流。

02

选择模型

按延迟、推理或编码需求选择合适的模型，然后接入你现有的客户端。

03

发送首个请求

请求跑通后，就可以在控制台里调整路由、用量和计费，而不用重写应用。
