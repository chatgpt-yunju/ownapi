# OpenClaw AI 用户控制台使用指南

## 问题修复

已修复 token 认证问题：
- 控制台现在会自动读取主站的 SSO token（存储在 `localStorage.token`）
- 如果主站 token 不存在，会尝试读取 `openclaw_token`
- 用户从主站登录后，可以直接访问控制台

## 访问控制台

### 方式 1：从主站跳转（推荐）

1. 访问 https://yunjunet.cn 并登录
2. 登录成功后，访问 https://api.yunjunet.cn/console.html
3. 控制台会自动使用主站的 SSO token 进行认证

### 方式 2：直接访问（需要先登录主站）

1. 确保已在 https://yunjunet.cn 登录
2. 直接访问 https://api.yunjunet.cn/console.html
3. 如果未登录，会自动跳转到主站登录页面

## 控制台功能

### 1. 仪表盘（概览）

显示用户的核心数据：
- **余额**: 当前账户余额（人民币）
- **今日调用**: 今天的 API 调用次数
- **今日 Tokens**: 今天使用的 token 数量
- **今日费用**: 今天的 API 调用费用

还包括：
- 近 7 天调用量图表
- 最近调用记录

### 2. API Keys（密钥管理）

管理你的 API 密钥：
- 查看所有 API 密钥
- 创建新的 API 密钥
- 启用/禁用密钥
- 删除密钥
- 查看密钥的创建时间和最后使用时间

**注意**：
- 每个用户最多可以创建 10 个 API 密钥
- 创建密钥时，完整的密钥只显示一次，请立即保存

### 3. 调用日志

查看 API 调用历史：
- 调用时间
- 使用的模型
- Token 使用量（输入/输出）
- 调用费用
- 调用状态（成功/失败）
- 调用 IP

支持筛选：
- 按模型筛选
- 按状态筛选（成功/失败）

### 4. 套餐

购买和管理套餐：

#### 可用套餐

| 套餐 | 价格 | 每日限额 | 月度配额 |
|------|------|----------|----------|
| Free | ¥0 | 20 次/天 | $0 |
| Pro | ¥129 | 5000 次/天 | $50 |
| Max | ¥499 | 20000 次/天 | $200 |
| Ultra | ¥1699 | 99999 次/天 | $800 |

#### 购买流程

1. 点击套餐的"购买"按钮
2. 系统会计算余额抵扣和需支付金额：
   - 如果余额充足：直接扣除余额，立即生成 API 密钥
   - 如果余额不足：使用余额抵扣部分金额，剩余部分通过支付宝支付
   - 如果余额为 0：全额通过支付宝支付
3. 支付完成后，系统会自动：
   - 创建 API 密钥
   - 关联套餐信息
   - 设置每日限额和月度配额
   - 设置套餐有效期（30 天）

#### 我的套餐

查看已购买的套餐：
- 套餐名称
- 开始时间
- 到期时间
- 状态（active/expired）

### 5. 调试（Playground）

在线测试 API 调用：
- 选择模型
- 输入消息
- 选择是否使用流式输出
- 发送请求并查看响应
- 查看调用统计（tokens、费用等）

## 支付说明

### 余额优先抵扣

购买套餐时，系统会自动使用余额抵扣：

**示例 1：余额充足**
- 套餐价格：¥129
- 当前余额：¥150
- 结果：扣除余额 ¥129，无需支付宝支付

**示例 2：余额不足**
- 套餐价格：¥129
- 当前余额：¥50
- 结果：扣除余额 ¥50，支付宝支付 ¥79

**示例 3：余额为 0**
- 套餐价格：¥129
- 当前余额：¥0
- 结果：支付宝支付 ¥129

### 支付宝支付流程

1. 点击购买后，系统生成支付订单
2. 自动打开支付宝支付页面
3. 扫码完成支付
4. 支付成功后，页面自动跳转回控制台
5. 系统自动创建 API 密钥并显示

### 订单查询

如果支付后没有自动跳转，可以：
1. 在控制台的"套餐"页面查看"我的套餐"
2. 在"API Keys"页面查看是否有新的密钥
3. 联系管理员查询订单状态

## API 使用

### 获取 API 密钥

1. 在控制台的"API Keys"页面创建密钥
2. 或者购买套餐后自动生成密钥
3. 复制完整的 API 密钥（格式：`sk-xxxxxxxx`）

### 调用示例

```bash
curl -X POST https://api.yunjunet.cn/v1/chat/completions \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-sonnet-4-6",
    "messages": [
      {"role": "user", "content": "你好，请介绍一下你自己"}
    ]
  }'
```

### 支持的模型

查看所有可用模型：
```bash
curl https://api.yunjunet.cn/api/models
```

常用模型：
- `claude-sonnet-4-6`: Claude Sonnet 4.6
- `claude-opus-4-6`: Claude Opus 4.6
- `deepseek-chat`: DeepSeek Chat
- `qwen-max`: 通义千问 Max
- 等等...

## 计费说明

### 按量计费

如果没有购买套餐，使用按量计费：
- 根据实际使用的 token 数量计费
- 费用从余额中扣除
- 不同模型价格不同（查看模型列表获取价格）

### 套餐计费

购买套餐后：
- 使用套餐的月度配额（美元）
- 受每日限额限制
- 套餐有效期 30 天
- 到期后自动失效

### 计费优先级

1. 优先使用套餐配额（如果有有效套餐）
2. 套餐配额用完或过期后，使用余额按量计费
3. 余额不足时，API 调用会失败

## 常见问题

### 1. 控制台显示"未登录"

**原因**：SSO token 过期或未登录主站

**解决方案**：
1. 访问 https://yunjunet.cn 重新登录
2. 登录成功后，再访问控制台

### 2. 菜单不显示

**原因**：浏览器缓存或 JavaScript 错误

**解决方案**：
1. 清除浏览器缓存
2. 按 Ctrl+F5 强制刷新页面
3. 检查浏览器控制台是否有错误

### 3. 支付后没有生成 API 密钥

**原因**：
- 支付回调未收到
- 用户已有 10 个密钥（达到上限）
- 订单状态未更新

**解决方案**：
1. 等待 1-2 分钟后刷新页面
2. 检查"API Keys"页面是否有新密钥
3. 检查是否已有 10 个密钥，如有请删除不用的密钥
4. 联系管理员查询订单状态

### 4. API 调用失败

**原因**：
- API 密钥错误
- 套餐已过期
- 每日限额用完
- 余额不足

**解决方案**：
1. 检查 API 密钥是否正确
2. 在控制台查看套餐状态
3. 查看今日调用次数是否超过限额
4. 检查余额是否充足

### 5. 余额显示不正确

**原因**：数据同步延迟

**解决方案**：
1. 刷新页面
2. 等待几秒后再查看
3. 如果仍然不正确，联系管理员
